# Max Sum Path in Grid

## 题目

给定 `N x M` 整数网格，从左上角出发，每步只能向右或向下，到达右下角时求路径元素最大和。`N,M <= 100`，元素范围 -1000 到 1000。

函数接口：`solution(...)`，直接返回题目要求的结果，不读取标准输入。

## 详细解析

一维 `dp[col]` 表示处理当前单元前从上方到达的最优和，`dp[col-1]` 是本行从左方到达的最优和；取两者较大值再加当前格。首格用 0 初始化，其余不可达方向为负无穷。

## 正确性说明

每个格子的最后一步必来自上方或左方，二者之外无合法来源。按行列顺序处理时两来源状态都已最优，取最大并加当前值便是当前格最优，归纳至右下角成立。

## 复杂度分析

时间 `O(NM)`，空间 `O(M)`。

## 样例与测试

原题全部可执行样例已写入 `test_solution.py`：3x3 网格 `[[1,2,3],[4,5,6],[7,8,9]] -> 29`；`[[-1,-2],[-3,-4]] -> -7`；题给 4x4 网格 `-> 73`。原文有的样例把 N、M 分两行，有的写同一行，函数接口不受影响。。测试直接调用函数接口，避免输入展示换行差异影响算法验证。
